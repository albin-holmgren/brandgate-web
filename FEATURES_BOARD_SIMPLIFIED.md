## 📋 SIMPLE STRUCTURE (3 Columns)

### ✅ To Do
Features waiting to be built  
**Add here when:**
- User requests a feature
- AI identifies critical gap
- Internal improvement needed

### 🔄 Doing
Features currently in development  
**Move here when:**
- Development starts
- Active work in progress

### ✅ Done
Features released to production  
**Move here when:**
- Feature is live
- Testing complete
- Users can access it

## 📝 COMPREHENSIVE TASK FORMAT

### Task Title Format:
```
🔴 HIGH / 🟡 MEDIUM / 🟢 LOW : Feature name (effort estimate)
```

**Examples:**
- 🔴 **HIGH:** Add SMS notifications for hot leads (3-5 days)
- 🟡 **MEDIUM:** Redesign dashboard with visual lead scoring (1-2 weeks)
- 🟢 **LOW:** Add AI-powered lead scoring optimization (2+ weeks)

---

## 📄 WHAT EACH TASK INCLUDES

### 1. **Header** (Quick Info)
```
PRIORITY: 🔴 HIGH | STATUS: To Do | ESTIMATE: 3-5 days
```

### 2. **Feature Description**
One-sentence summary of what this feature does.

### 3. **Why Build This** (Business Case)
- Problem it solves
- Who benefits
- What happens if we don't build it

### 4. **User Impact**
- Critical / High / Medium / Low
- How many users affected

### 5. **Category**
- UI/UX, Backend, Integration, Performance, Security, Mobile, Analytics

### 6. **Business Value**
- 💰 Revenue Generating
- 💵 Cost Saving
- 🔄 User Retention
- 🏆 Competitive Advantage

### 7. **Acceptance Criteria** (Checklist)
Specific, testable requirements:
```
□ Integrate with SMS provider
□ Send SMS when lead score > 85
□ Include opt-out instructions
□ Rate limit: max 1 SMS per day
```

### 8. **Technical Notes**
- Implementation approach
- Tools/libraries to use
- Architecture considerations

### 9. **Success Metrics**
Measurable outcomes:
- Response time < 30 minutes
- User satisfaction > 4/5
- Zero spam complaints

### 10. **Requested By**
- Customer name
- "AI Analysis" (if I identified it)
- "Internal" (if it's our idea)

## 🎯 PRIORITY GUIDE

### 🔴 HIGH Priority
**Build first if:**
- Critical bug or security issue
- Blocking other work
- Direct revenue impact
- Competitive necessity
- Users asking urgently

**Timeline:** 1-2 weeks max

### 🟡 MEDIUM Priority
**Build next if:**
- Significant user benefit
- Improves key metrics
- Good ROI
- High user demand

**Timeline:** 1-2 months

### 🟢 LOW Priority
**Build later if:**
- Nice improvement
- Experimental/unknown value
- Limited impact
- "Maybe someday"

**Timeline:** When time permits

## 🔄 WHEN TO ADD FEATURES

### ✅ Add to "To Do":
1. **User specifically requests**
   - Capture exact need
   - Ask why it's important
   - Understand use case

2. **AI identifies critical gap**
   - Workflow inefficiency
   - Missing functionality
   - User pain point

3. **Internal strategic need**
   - Competitive response
   - Technical debt
   - Scale requirement

### ❌ DON'T Add:
- Daily operations (use Operations board)
- Marketing tasks
- Content creation
- Vague ideas without clear value
